rAdam23 Website

Full-stack Web Design

1. The Website

rAdam23 is a portfolio website that showcases all my portfolio work and across design and art in one hub.

2. The Brief

To design and hand code a fully responsive portfolio website end to end using my current skill set (UX, UI, Graphic Design, HTML, CSS). To deploy and launch the website and update it by Git.

3. My Role

Full-Stack Web Designer

4. The Website

Visually clean in design, the website has been hand-coded to be fully responsive using HTML5 and CSS3.

The custom code enables the site to run very lightweight and fast, providing a better user experience to visiters with minimal load times.

1. Defining the goal

I needed a custom built website for my rAdam23 design portfolio. The site needed to contain my case studies, be fully responsive, coded by hand (to ensure speed) and contain as many custom assets as possible.

2. Moodboards

Inspiration for the look and feel of the website was drawn from lifestyle website Broadsheet and the New Yorker magazine. I wanted a clean look and feel to the website that was modern, contemporary and worked with a responsive design.

3. Identifying user flows

I started mapping the existing userflows using inspiration from Broadsheet and the New Yorker website layouts. I decided here that nav menus shouldnt be overly nested and preferred to more clearly list links for navigational context and ease of use.

4. Designing the wireframes

Wireframes were designed with widescreen desktop, desktop, tablet and mobile screen sizes in mind. The design was not dependant on device size breakpoints as I wanted the design to accommodate all the types of screen sizes out there today.

5. Coding the front end

HTML5 and CSS3 was used for the frontend. The layout worked best with grids so I decided to code this in as it offered a clean solution to my design and responsive requirements.

6. Testing responsiveness

Because of the importance in having a good user experience for all visitors to the website, it was crucial that all areas of the site were visually appealing across all screen sizes. Although relatively simple in principle and code, this took some time to design as all sections and areas had to be tested individually to ensure each area of the design accepted all configurations.

7. The Final Prototype

While defining userflows and the feel of the project I played around with layouts, experimenting with conventional banking platform layouts. I settled on a card layout design as it fit the brief, clearly defining information in containers, whilst also being exptremely adaptable and responsive to all types of mobile devices today.
